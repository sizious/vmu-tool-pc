7. Managing invalid files

This program can easilly manage invalid Visual Memory datas. For example, missing VMI files. At this point, it can repair this kind of errors, but maybe in the future, repair corrupted datas...

Example :

1) Managing invalid VMI

If you have autorized the display of errornous VMI files (see config section), you should see theses files in the view :

At this point, nothing can be done with invalid VMI files. In fact, VMI files are only VMS header files. For exemple, as header (.h) files for source files (.c). You can check the debug tab to see what kind of error it is.

2) Managing invalid VMS

Invalid VMS files are only displayed if you have checked the option in the config section. Theses files seems to be valid, but are not read because no VMI associated file was found. Example :

To solve this problem, right click on the errornous VMS :

A new dialog is shown :

You can put the mouse cursor on a control to see a little description hint.

At this point, you can change the Description, Copyright fields if you want. Please note that field, DC Filename, is required. DC Filename's the name of the file when it transfered to the Dreamcast. Example : SONICADV_INT for Sonic Adventure 1. This field must be filled carefully. If not, the file won't be recognized on the Dreamcast .

Click Create... button to save the new VMI file. The VMI file must be in the same directory of the VMS.

The view should be refreshed. That's all ! You done.
